# googo.io

## 说明

```
本人有多年golang、php、nodejs、前端开发经验，在n多个项目中不断提炼整理自己的框架思维。
上学时学习java的Struts+Spring+Hibernate，被称之为最为流行的MVC。
上班后被迫转向php，从学习ci框架，阅读ci源码，到深刻理解MVC，自己学着写PHP框架，终于弄懂浏览器到服务器，服务器到程序代码的原理(茅塞顿开)。
后来工作中又接触了nodejs，发现所谓的MVC，不论语言，大体思路、代码结构基本都如出一辙；
随着互联网的发展，技术的发展，应用系统已经不满足简简单单的MVC了。
我对技术的探索有着一种深深的渴望，一直想寻求到更好的开发模式，代码结构。
从单体架构、到分布式架构、再到微服务架构；从MVC，到MC，再到MSC的开发模式演变中，我总结出以下两点：
1. 程序要拆分，设计要分层
2. 请求 -> 网关 -> 业务 -> 数据
```
